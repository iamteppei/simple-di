# Security Policy

## Supported Versions

The project is currently in active prototype phase.

Security fixes are applied on a best-effort basis to the latest code in the
default branch.

## Reporting a Vulnerability

Please report security vulnerabilities privately.

Do not open a public issue for suspected vulnerabilities.

Use one of the following private channels:

- tamnguyen1560@gmail.com

When reporting, include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Impact assessment
- Suggested remediation, if known

## Response Process

After receiving a report, maintainers will:

1. Acknowledge receipt as soon as possible.
2. Triage and validate the report.
3. Work on a fix and coordinate disclosure timing.
4. Publish remediation details after a fix is available.

## Disclosure Policy

We follow coordinated disclosure.

Please allow maintainers reasonable time to investigate and release a fix before
publicly disclosing details.
